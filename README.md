# LLM Diagnostic Accuracy Analysis on the Trustworthy AI Psychotherapy Dataset

A reproducibility and error-analysis study evaluating how four LLMs — **Llama-4**, **Mistral-Saba**, **Qwen-QWQ**, and **GPT-4.1-Nano** — perform on simulated mental health diagnosis tasks, based on the open-source dataset and multi-agent workflow from *"Trustworthy AI Psychotherapy: Multi-Agent LLM Workflow for Counseling and Explainable Mental Disorder Diagnosis"* (Ozgun et al., CIKM '25).

This project reproduces the paper's **Table 3** (per-disorder F1 by model) directly from the raw conversation logs, extends it with full classification reports, confusion matrices, misdiagnosis pattern analysis, an error network graph, statistical significance testing, and a clinical-risk framing of the most dangerous misclassification pathways.

## Project Overview

Large language models are increasingly explored as triage assistants, screening tools, and conversational "AI therapists." This project asks a simple but important question: **when an LLM-based multi-agent workflow conducts a simulated DSM-5 diagnostic interview, how often does it arrive at the correct diagnosis — and when it doesn't, what does it confuse the diagnosis with?**

The analysis covers:

- Global accuracy, macro/weighted precision, recall, and F1 for each model
- Per-disorder precision, recall, F1, and support (10 DSM-5 categories)
- A direct reproduction of the paper's published Table 3
- Raw and normalized confusion matrices for each model
- Dominant misdiagnosis pattern per disorder per model
- A directed error-network graph of the top 20 confusion pathways
- McNemar's significance testing and bootstrap confidence intervals
- A clinical-risk ranking of high-stakes misclassification pathways (e.g., Bipolar → Depression)

## Dataset

**Source:** [`github.com/mithatco/mental_health_multiagent`](https://github.com/mithatco/mental_health_multiagent) — the open-sourced implementation accompanying Ozgun et al. (CIKM '25).

- **~8,000 simulated therapist–client conversations** (~2,000 per model), generated by a three-agent LLM workflow (therapist agent, client agent, diagnostician agent).
- Each simulated client is assigned **one of 10 primary disorder profiles**: Adjustment, Anxiety, Bipolar, Depression, OCD, Panic, PTSD, Schizophrenia, Social Anxiety, Substance Abuse.
- After the interview, a **diagnostician agent** produces a narrative diagnosis grounded in retrieved DSM-5 passages (RAG).
- An **evaluator LLM** then classifies that narrative into one of the same 10 categories (or `"other"`) for comparison against the assigned profile.

There is no packaged HuggingFace dataset for this paper — the notebook clones the GitHub repository directly (~350MB, shallow clone) and parses every JSON conversation log.

> **Important:** each model was evaluated on its **own independent sample** of ~2,000 conversations — not a shared, fixed test set. This affects how paired statistical tests (McNemar's) are constructed (see [Methodology](#methodology)).

## Methodology

1. **Data loading** — shallow `git clone` of the source repository; parse `metadata.patient_profile` (ground truth) and `evaluation.llm_evaluation.diagnosis_classification.classified_as` (prediction) from each of the ~8,000 JSON logs.
2. **Cleaning** — label normalization (case/whitespace/underscore handling), text cleanup, and detection of infrastructure-level API errors (see [below](#data-quality-note-apiinfrastructure-errors)).
3. **Global evaluation** — accuracy, macro precision/recall/F1, weighted F1 per model.
4. **Per-disorder evaluation** — full `classification_report` per model, pivoted into a Table-3-style matrix (Disorder × Model F1, with Mean F1 and Best Model columns).
5. **Confusion analysis** — raw and row-normalized confusion matrices; for each disorder, the single most common incorrect prediction and what % of that disorder's errors it represents.
6. **Error network** — directed graph of the top 20 (True → Predicted) misclassification pairs aggregated across all four models, visualized with `networkx`.
7. **Statistical testing**:
   - **McNemar's test** — since the four models were not evaluated on a shared fixed test set, paired observations are constructed by **randomly matching cases with the same ground-truth label** across model pairs (without replacement, up to `min(n_A, n_B)` per class). This is an approximation of a true paired design under the assumption that within-class difficulty is comparable across randomly drawn simulated clients.
   - **Bootstrap 95% confidence intervals** for accuracy, computed independently per model (valid regardless of pairing).
8. **Clinical risk analysis** — a curated set of high-stakes confusion pathways (e.g., Bipolar → Depression, Substance Abuse → Depression) is cross-referenced against the aggregated confusion-pair frequencies.

All code is modular, type-hinted, and documented with docstrings; metrics use vectorized `scikit-learn` operations.

## Key Results

### Global Performance

*(Computed with `EXCLUDE_API_ERRORS = True` — see [Data Quality Note](#data-quality-note-apiinfrastructure-errors))*

| Model | Accuracy | Macro Precision | Macro Recall | Macro F1 | Weighted F1 |
|---|---|---|---|---|---|
| **Qwen-QWQ** | **0.7945** | 0.8034 | **0.7268** | **0.7486** | **0.8162** |
| GPT-4.1-Nano | 0.7210 | **0.8301** | 0.6566 | 0.6665 | 0.7349 |
| Llama-4 | 0.6348 | 0.7430 | 0.5793 | 0.5846 | 0.6347 |
| Mistral-Saba | 0.5954 | 0.7450 | 0.5414 | 0.5609 | 0.6139 |

**Qwen-QWQ** — the only model among the four specifically designed for step-by-step reasoning rather than conversational fluency — achieves the best overall scores on every metric except Macro Precision, where **GPT-4.1-Nano** leads (0.8301). This mirrors the paper's central finding: reasoning capability appears more important than conversational quality for diagnostic accuracy.

### Misdiagnosis & Clinical Risk

The single most consistent failure mode across **all four models** is:

> **Adjustment Disorder → predicted as Depression**

Three of the four models (Llama-4, Mistral-Saba, GPT-4.1-Nano) score below **3% F1** on Adjustment Disorder, indicating they almost never correctly identify it — they default to Depression instead. Only Qwen-QWQ achieves a moderate 40.4%.

Other notable confusion pathways flagged as **clinically high-risk** (i.e., carrying real treatment implications if acted upon):

| Pathway | Clinical Risk |
|---|---|
| Bipolar → Depression | Risk of antidepressant monotherapy triggering manic episodes if bipolar disorder is missed |
| Substance Abuse → Depression | Risk of masking a critical comorbidity, leading to incomplete treatment |
| Social Anxiety → Anxiety | Risk of generic anxiety treatment that misses social-specific exposure needs |
| PTSD → Anxiety | Risk of missing trauma-focused interventions (e.g., trauma-focused CBT, EMDR) |
| Schizophrenia → Bipolar | Risk of delaying antipsychotic treatment |

## Key Findings

- **Easiest disorder to diagnose:** **PTSD** (mean F1 ≈ 93.6%) — the most distinctive presentation across all four models.
- **Hardest disorder to diagnose:** **Adjustment Disorder** (mean F1 ≈ 11.8%) — a *systemic* weakness affecting every model, driven by symptom overlap with Depression.
- **Best model overall:** **Qwen-QWQ** (Macro F1 ≈ 0.749) — the reasoning-optimized model outperforms three conversationally-fluent models on diagnostic accuracy.
- **Widest model-specific gap:** **Bipolar Disorder** — Llama-4 (46.4% F1) vs. Qwen-QWQ (96.1% F1), a >2x difference suggesting reasoning depth, not just model size, drives differentiation of manic/depressive episode patterns.
- **Depression as a "default" label:** Depression's moderate F1 (37–69%) despite being one of the most prevalent and well-studied categories suggests models over-predict it as a catch-all when uncertain — depressing its precision even as recall stays relatively high.

## Data Quality Note: API/Infrastructure Errors

During data cleaning, a subset of conversation logs were found to contain **Groq API failures** (`"Error communicating with Groq: 400 Client Error..."`) instead of an actual diagnostician narrative. The evaluator LLM still assigns these a label (typically `"other"`), meaning by default they are counted as **diagnostic errors** — conflating infrastructure failures with genuine model misdiagnoses.

| Model | API Errors / Total | % |
|---|---|---|
| Llama-4 | 56 / 2000 | 2.80% |
| Qwen-QWQ | 24 / 2000 | 1.20% |
| Mistral-Saba | 13 / 2000 | 0.65% |
| GPT-4.1-Nano | 0 / 2000 | 0.00% |

The notebook includes a toggle, `EXCLUDE_API_ERRORS` (default `True`), which removes these 93 rows before computing any metric. All results in this README reflect `EXCLUDE_API_ERRORS = True`. Setting it to `False` reproduces the paper's published Table 3 essentially exactly (differences of ≤1.5 points per cell, likely due to evaluator-run variance).

## Running the Notebook

The notebook is designed for **Google Colab** and runs top-to-bottom with no manual configuration:

1. Open `LLM_Diagnostic_Accuracy_Analysis_Trustworthy_AI_Psychotherapy.ipynb` in Colab.
2. Run all cells (`Runtime → Run all`).
3. Section 3 will shallow-clone the source dataset repository (~350MB, ~1-2 minutes). If the clone fails for any reason, a synthetic dataset generator — calibrated to the paper's published Table 3 F1 scores — is used automatically as a fallback, so every downstream cell remains executable.
4. All output tables and figures render inline; CSV artifacts are saved to `outputs/` and offered as Colab downloads in the final section.

**Dependencies** (installed automatically in the first cell): `pandas`, `numpy`, `matplotlib`, `seaborn`, `scikit-learn`, `scipy`, `plotly`, `networkx`.

## Limitations & Future Work

- **Not a shared test set.** Each model was evaluated on its own ~2,000-conversation sample, not an identical fixed set of clients. McNemar's test uses label-matched random pairing as an approximation; a future shared-test-set release would enable a true paired comparison.
- **Single-label evaluation.** The dataset's simulated client profiles include comorbid modifiers that this single-label evaluation discards. Multi-label evaluation would better reflect real diagnostic complexity.
- **"Other" classification bucket.** A small fraction of evaluator-LLM classifications fall into `"other"`, which may itself be a signal of diagnostician-agent output quality worth investigating separately.
- **Reasoning vs. fluency.** Qwen-QWQ's advantage on Adjustment Disorder and Bipolar Disorder suggests step-by-step reasoning capability materially improves diagnostic differentiation — a hypothesis worth testing directly by comparing reasoning vs. non-reasoning variants of the same base model.
- **Potential extensions:** fine-tuning or prompt-engineering targeted at Adjustment Disorder / Depression differentiation (the dominant systemic weak point); a traditional ML baseline (TF-IDF + logistic regression) on the diagnostician narratives for comparison against the LLMs.
